Corrupted mysql¶

Restoring¶

  1. create new blank database

  2. copy dtp API instance and create a temporary instance

  3. modify new instance to use new database

  4. rebuild schema (run framework’s database migration scrip)

  5. import lost table data to new database (take from old backups where data exist)

  6. create msqldump of up-to-date data

  7. import up-to-date data

  8. test

  9. switch production database to new one

  10. remove temporary instance
